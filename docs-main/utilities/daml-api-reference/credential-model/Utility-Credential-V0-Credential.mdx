# Utility.Credential.V0.Credential

## Templates

<a id="type-utility-credential-v0-credential-credential-66032"></a>
**template** [Credential](/docs-main/utilities/daml-api-reference/credential-model/Utility-Credential-V0-Credential#type-utility-credential-v0-credential-credential-66032)

> A credential contract issued by an *issuer* to a *holder* for a specific purpose. It allows an
> issuer to attest a fact about a *subject*. The subject is typically the same as the holder of the
> credential.
> Credentials should be *immutable*, so that they can be fetched (i.e. verified) without
> contention.
> Credentials can be configured to be *existential*: by not providing an expiry date, they are
> valid as long as the contract exists on ledger.
>
> Signatory: issuer, holder

**data** [PartyCredentialRequirement](/docs-main/utilities/daml-api-reference/credential-model/Utility-Credential-V0-Credential#type-utility-credential-v0-credential-partycredentialrequirement-96395)

> Defines the criteria a credential must satisfy to prove authorization for a `Party` (the
> `subject`). The validation will additionally check that the credential is held by either the
> `subject` or the `issuer`.
>
> NOTE: This data type is used internally by Utility applications. External consumers should use
> their own credential requirement definitions (as these might evolve independently).
>
