# JSON Ledger API references

You can use [openapi.yaml](openapi.mdx#reference-json-api-openapi) and [asyncapi.yaml](asyncapi.mdx#reference-json-api-asyncapi) files that describe the JSON Ledger API and the streaming API, respectively.

## Location of JSON Ledger API files

If you have a running instance of Canton with JSON Ledger API enabled, you can download definitions using links:

* **http://&lt;host&gt;:&lt;port&gt;/docs/openapi**
* **http://&lt;host&gt;:&lt;port&gt;/docs/asyncapi**

where **&lt;host&gt;** and **&lt;port&gt;** are the host and port of your Canton instance. Check  [configuration](../../explanations/json-api/index.mdx#reference-json-api-configuration).

For a general introduction to the JSON Ledger API , see the following sections [JSON Ledger API Service V2](../../explanations/json-api/index.mdx#json-api).

## Daml values

Some fields (for instance createArguments in CreateCommand) do not specify any type.

This means the actual type is determined by the Daml model. See [Daml type conversion](lf-value-specification.mdx#reference-json-lf-value-specification) for more details.
