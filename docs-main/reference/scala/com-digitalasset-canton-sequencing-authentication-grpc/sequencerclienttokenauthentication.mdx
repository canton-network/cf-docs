---
title: "SequencerClientTokenAuthentication"
description: "Will invalidate the current token if an UNAUTHORIZED response is observed. This will typically happen after a token has expired. Note the caller will still receive the UNAUTHORIZED response, although there are approaches for buffering and retrying the request this would not work for all cases (such as a streamed response). Instead, the caller is expected to retry the request which will attempt to fetch a new authorization token as the prior response invalidated the previous token."
---
