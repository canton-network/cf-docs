## Development Stack Overview

Canton development involves these components:

```mermaid
flowchart TB
    subgraph YourCode[Your Application]
        FE[Frontend<br>React, Vue, etc.]
        BE[Backend<br>TypeScript, Java]
        SC[Smart Contracts<br>Daml]
    end

    subgraph Canton[Canton Infrastructure]
        PART[Participant Node]
        PQS[PQS<br>SQL queries]
        SYNC[Synchronizer]
    end

    FE --> BE
    BE --> PART
    BE --> PQS
    PART <--> SYNC
    SC --> PART
```

## Prerequisites

Before starting development:

### Required

- **Programming experience** in any language
- **Command line** familiarity
- **Git** for version control

### Helpful (but not required)

- **Functional programming** concepts (Haskell, OCaml, F#, or similar)
- **Docker** for running local environments
- **PostgreSQL** for PQS queries

## QuickStart: The Fastest Path

If you want to see Canton in action immediately:

1. **Clone the QuickStart repository:**
   ```bash
   git clone https://github.com/digital-asset/cn-quickstart
   cd cn-quickstart
   ```

2. **Set up and start the local environment:**
   ```bash
   make setup
   make build
   make start
   ```

3. **Explore the demo application**

The QuickStart includes:
- Working Daml contracts
- Java backend
- React frontend
- LocalNet configuration

<Note>
The QuickStart requires Docker with at least 8GB RAM allocated. See [QuickStart Prerequisites](/docs-main/appdev/quickstart/prerequisites) for detailed requirements.
</Note>
