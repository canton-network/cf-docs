---
title: "LookupSentAcsCommitmentsResponse"
description: "Returns a sequence of commitments for each synchronizer. Synchronizers should not repeat in the response, otherwise the caller considers the response invalid. If all commitments sent on a synchronizer have been pruned, we return an error. No streaming, because a response with verbose mode on contains around 1kb to 3kb of data (depending on whether we ship the LtHash16 bytes directly or just a hash thereof). Therefore, 1000 commitments fit into a couple of MBs, and we can expect the gRPC admin API to handle messages of a couple of MBs. It is the application developer's job to find suitable filter ranges."
---
