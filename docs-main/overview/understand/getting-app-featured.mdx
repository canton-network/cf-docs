As the Canton Network ecosystem grows, there are opportunities to get your application featured and discovered by users. This page explains how.

## Where Apps Are Featured

| Platform | Description |
|----------|-------------|
| [canton.network](https://canton.network) | Main network website |
| [sync.global](https://sync.global) | Global Synchronizer Foundation |
| Ecosystem directories | Partner listings |
| Community channels | Developer showcases |

## Requirements for Featuring

To be considered for featuring, your application should satisfy the following points:

### Technical Requirements

| Requirement | Description |
|-------------|-------------|
| **Deployed on Canton Network** | Running on DevNet, TestNet, or MainNet |
| **Functional** | Working application (not just a concept) |
| **Follows standards** | Implements relevant CIPs (e.g., token standard) |
| **Security reviewed** | Basic security practices followed |

### Documentation Requirements

| Requirement | Description |
|-------------|-------------|
| **User documentation** | How to use the application |
| **Technical overview** | What the application does |
| **Support channels** | How users get help |

### Quality Expectations

| Aspect | Expectation |
|--------|-------------|
| **User experience** | Reasonable usability |
| **Stability** | Reliable operation |
| **Transparency** | Clear about capabilities and limitations |
| **Support** | Responsive to user issues |

## Application Process

### For Canton.network Listing

1. **Complete development** and deploy your application
2. **Prepare materials**:
   - Application description (short and long)
   - Logo and screenshots
   - Contact information
   - Documentation links
3. **Contact** the Canton Network team via [canton.network](https://canton.network)
4. **Review process** (timeline varies)
5. **Listing** upon approval

### For Ecosystem Partner Directories

Individual ecosystem partners may have their own listing processes. Check with specific partners for their requirements.

## Promotional Opportunities

### Community Showcase

Present your application to the developer community:

| Opportunity | Description |
|-------------|-------------|
| **Office hours** | Demo during community calls |
| **Slack** | Share in appropriate channels |
| **Forum posts** | Technical write-ups |
| **Blog posts** | Announcements and updates |

### Events

Present your application during events:

| Event Type | Opportunity |
|------------|-------------|
| **Developer workshops** | Demo or presentation |
| **Ecosystem events** | Sponsor or exhibit |
| **Conferences** | Canton Network presence |

## Best Practices

### Building for Discovery

| Practice | Benefit |
|----------|---------|
| **Solve real problems** | Users seeking solutions will find you |
| **Integrate with ecosystem** | Appear in partner listings |
| **Follow standards** | Wallet and tool compatibility |
| **Document well** | Users can onboard easily |

### Maintaining Presence

| Activity | Frequency |
|----------|-----------|
| **Update listings** | When features change |
| **Engage community** | Regularly |
| **Respond to users** | Promptly |
| **Share updates** | On major releases |

## What to Avoid

| Avoid | Why |
|-------|-----|
| **Misleading claims** | Damages trust |
| **Abandoned projects** | Frustrates users |
| **Poor security** | Risks users and reputation |
| **Spam promotion** | Violates community norms |
