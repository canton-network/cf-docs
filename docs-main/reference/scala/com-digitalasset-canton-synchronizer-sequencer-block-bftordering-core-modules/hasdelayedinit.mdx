---
title: "HasDelayedInit"
description: "Called when a module completes relevant initialization and is ready to resume processing events. Any postponed events are dequeued and processed using the provided message handler. For clean startup, we recommend only calling method once per module. However, this method is still idempotent, as subsequent invocations should have no effect."
