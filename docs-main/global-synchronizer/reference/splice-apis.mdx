The Splice APIs are REST APIs exposed by the Canton Network application layer (Scan and Validator apps). They provide access to network state, wallet operations, and administrative functions beyond what the standard Ledger API offers.

## Scan API

The Scan App is part of every Super Validator (SV) node. It stores the transaction history and active contract set as seen by SVs, along with reference data such as synchronizer connection details and package information.

The Scan API provides access to this data for third-party integrations such as network analytics apps, block explorers, and financial accounting systems.

The Scan App subscribes to the SV participant as the DSO (Decentralized Synchronizer Operations) party. It observes Canton Coin transfers and governance operations but does not see custom Daml app transactions that don't interact with Canton Coin.

### Sub-APIs

- **Bulk Data API** — Full history of updates and ACS snapshots from the SV participant
- **Global Synchronizer Connectivity API** — Connection information for the Global Synchronizer
- **Global Synchronizer Operations API** — Operational state of the synchronizer
- **Canton Coin Reference Data API** — Canton Coin configuration and parameters
- **Current State API** — Current active contract state
- **Aggregates API** — Aggregated metrics and statistics

### BFT Reads

You can connect directly to a single SV's Scan API.  However, it is recommended to do a BFT read using the `scan-proxy`. The validator node includes a scan proxy (`/v0/scan-proxy/...`) that provides BFT reads by broadcasting to multiple SV scan services and returning a consensus result.

The Validator App is part of every validator node. It manages the Canton participant, automates core Canton Network Daml workflows, and exposes REST APIs for wallet and node operations.

### API Groups

| API | Purpose | Stability |
|-----|---------|-----------|
| **User Wallet API** | Users managing their wallets | External (stable) |
| **External Signing API** | External party Canton Coin operations | Internal |
| **User Management API** | Managing hosted users | Internal |
| **Validator Management API** | Node operator administration | Internal |
| **ANS API** | Amulet Name Service operations | External (stable) |
| **Scan Proxy API** | BFT proxy to SV scan services | See Scan API |

## Authentication

All Splice APIs require JWT authentication. Obtain tokens from your OIDC provider. The required claims depend on the specific API endpoint — user wallet endpoints require a token with the user's subject claim, while admin endpoints require the validator operator's subject claim.
