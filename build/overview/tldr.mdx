# TL;DR for new Canton Network developers

Use this document to begin your journey as a Canton Network developer.
We recommend you join the suggested Slack channels,
then work through the introductory links under each subheading to orient yourself to Daml and the Canton Network,
then begin building as early as possible. Return to this guide whenever you hit a roadblock.

## Installing the SDK

If you are working on the latest `Splice 0.5.0` / `Canton 3.4` project then we recommend to follow the [Digital Asset Package Manager (Dpm)](../dpm/dpm.mdx#dpm) instructions.

If you are working on a `Splice 0.4.x` / `Canton 3.3` project then continue to use the daml-assistant `Daml Assistant`.

Alternatively, use the quick start approach to quickly install the Daml SDK.

- [Install the Daml SDK](https://docs.digitalasset.com/build/3.3/quickstart/download/cnqs-installation.html#install-daml-sdk)

## Canton Network APIs

The following APIs are helpful in building a Canton Network dApp.

- [Validtor Ledger API](https://docs.sync.global/app_dev/ledger_api/index.html#app-dev-ledger-api)
- Canton Network [Scan API](https://docs.sync.global/app_dev/scan_api/index.html#app-dev-scan-api)
- Canton Network [Validator API](https://docs.sync.global/app_dev/validator_api/index.html#app-dev-validator-api)
- CIP-0056 [Token Standard APIs](https://docs.sync.global/app_dev/cip-0056/index.html#cip-0056-token-standard-apis)
- [dApp](https://github.com/hyperledger-labs/splice-wallet-kernel/blob/feed4e6165d07e6329f9ce431147c5024ae99431/api-specs/openrpc-dapp-api.json#L1) API (new)

Wallet developers should review the [Wallet SDK](https://docs.digitalasset.com/integrate/devnet/integrating-with-canton-network/index.html#how-to-install-the-wallet-sdk).

Use the [JSON API](https://docs.digitalasset.com/build/3.3/reference/json-api/openapi.html) to create language bindings for the Ledger API (LAPI) from the OpenAPI spec.
[Daml-LF JSON encoding](../reference/json-api/lf-value-specification.mdx#reference-json-lf-value-specification) describes the Daml &lt;-&gt; JSON encodings received from the payloads.
