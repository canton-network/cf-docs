## Version 0.5.9

### Scan

- Added `canton.scan-apps.scan-app.acs-store-descriptor-user-version` and `canton.scan-apps.scan-app.tx-log-store-descriptor-user-version` configuration settings to set a `user-version` for the ACS and TxLog stores respectively. Modifying the `user-version` wipes the respective store and triggers re-ingestion.

- Added a new external endpoint `GET /v0/unclaimed-development-fund-coupons` to retrieve all active unclaimed development fund coupon contracts.

### Daml

- Implemented minting delegation as described in [CIP-0073 — Weighted Validator Liveness Rewards for SV-Chosen Parties](https://github.com/global-synchronizer-foundation/cips/blob/main/cip-0073/cip-0073.md).

  New templates added in the `splice-wallet` package:

  - **MintingDelegationProposal** — Represents a proposal from beneficiary to the delegate to create a `MintingDelegation`
  - **MintingDelegation** — Represents an active delegation granting the delegate the authority to mint rewards on behalf of the beneficiary

  Enabling this feature requires SVs to vote on wallet package version `0.1.16`.

### Wallet

- Added endpoint `POST /v0/wallet/development-fund-coupons/allocate` to allocate a development fund coupon
- Added endpoint `GET /v0/wallet/development-fund-coupons` to retrieve active DevelopmentFundCoupon contracts
- Added endpoint `POST /v0/wallet/development-fund-coupons/{contract_id}/withdraw` to withdraw a development fund coupon
- Added minting delegation management to the wallet UI
- Enhanced rewards collection automation to support collecting development fund coupons

### Validator

- Added automation to perform minting for hosted external parties with an active `MintingDelegation` contract with an internal party

<Warning>
Version 0.5.7 introduced a significant performance regression related to processing topology transactions. Skip 0.5.7 and upgrade directly to 0.5.8.
</Warning>

### Canton

- Fixed performance regression related to processing topology transactions
- Improved performance of queries used in participant pruning (database migration may take up to 2 minutes)

### Scan

- Removed deprecated `/v0/total-amulet-balance` and `/v0/wallet-balance` endpoints

### Deployments

- Default logger switched to asynchronous appender for better performance. Set `LOG_IMMEDIATE_FLUSH=true` to revert to synchronous logging.
- This version breaks backwards compatibility with migration dumps from 0.4.x. Deploy with `migrating: false` (Helm) or without `-M` (Docker Compose).

## Upgrade Notes

Before upgrading to any new release:

1. Read the full release notes for breaking changes and required configuration updates
2. Back up all databases and node identities (see [Backup and Recovery](/docs-main/global-synchronizer/production-operations/backup-and-recovery))
3. Test the upgrade on DevNet or TestNet before applying to MainNet
4. For major upgrades requiring downtime, follow the [Upgrade Procedures](/docs-main/global-synchronizer/production-operations/upgrade-procedures)
