The Ledger API is the primary programmatic interface to a Canton participant node. Applications use it to submit commands (create contracts, exercise choices), read the transaction stream, and manage parties and packages.

## Quick Reference

| Property | Value |
|----------|-------|
| **Protocol** | gRPC (HTTP/2 + Protocol Buffers) |
| **Default port** | 5001 |
| **API version** | v2 (Canton 3.x) |
| **Authentication** | JWT (RS256) via OIDC |

## SDK Access Patterns

### TypeScript / JavaScript

Use the JSON API (HTTP wrapper around gRPC) for browser and Node.js applications:

```bash
# JSON API default port
curl http://localhost:7575/v2/version
```

The Wallet SDK provides higher-level abstractions over the JSON API for Canton Coin operations and the Token Standard.

### Java / Scala

Use the gRPC bindings directly. Generate type-safe Java code from your Daml models:

```bash
dpm codegen-java
```

The generated bindings include contract classes, choice wrappers, and event decoders.

### Code Generation

DPM generates language-specific bindings from your compiled Daml models:

```bash
# Java bindings
dpm codegen-java

# JavaScript/TypeScript bindings
dpm codegen-js
```

## Key Services

- **CommandService** — Submit commands and wait for completion (synchronous)
- **UpdateService** — Stream ledger updates (transactions, reassignments)
- **StateService** — Query the active contract set
- **InteractiveSubmissionService** — Prepare transactions for external signing

## Proto Definitions

gRPC proto files for the Ledger API are available in the [Canton repository](https://github.com/digital-asset/canton/tree/main/community/ledger-api/src/main/protobuf/com/daml/ledger/api/v2).
