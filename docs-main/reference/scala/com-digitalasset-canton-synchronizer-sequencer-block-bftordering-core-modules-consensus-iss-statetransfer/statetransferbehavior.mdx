---
title: "StateTransferBehavior"
description: "A state transfer behavior for IssConsensusModule. There are 2 types of state transfer: onboarding (for new nodes) and catch-up (for lagging-behind nodes). These two types work similarly, differing only in the triggering mechanism. State transfer requests commit certificates from a single node at a time and advances the topology accordingly, supporting multiple topology changes throughout the process. Thus, it depends on historical topology state from the latest pruning point. Moreover, it might use keys that are currently revoked (and even compromised) in signature verification during at least part of the process. The latter limitation is acceptable because key rotation, combined with regular pruning, generally prevents compromised keys in Canton."
---
