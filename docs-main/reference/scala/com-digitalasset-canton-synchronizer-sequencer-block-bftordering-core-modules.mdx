## `HasDelayedInit`

- Introduced: `3.4.10`
- Deprecated: -
- Removed: -

Upstream docs: [Open](https://docs.digitalasset.com/operate/3.4/scaladoc/com/digitalasset/canton/synchronizer/sequencer/block/bftordering/core/modules/HasDelayedInit.html)

**Signature**

```text
trait HasDelayedInit[M] extends AnyRef
```

**Summary**

Called when a module completes relevant initialization and is ready to resume processing events. Any postponed events are dequeued and processed using the provided message handler. For clean startup, we recommend only calling method once per module. However, this method is still idempotent, as subsequent invocations should have no effect.
