title: "Decentralized Attestation"
description: "Proving facts about off-chain state on the Canton Network using decentralized attestation and vaults"
---

Decentralized attestation is a mechanism for proving facts about off-chain state within Canton Network transactions. It bridges the gap between external data sources and on-ledger workflows.

## The Problem

Many real-world applications need to reference data that originates outside the ledger: identity credentials, market prices, regulatory status, or the state of assets on other networks. Today, most blockchain systems handle this through centralized oracles -- a single entity attests that a fact is true, and the ledger trusts that entity.

Centralized oracles create a single point of failure and a trust bottleneck. If the oracle is compromised or dishonest, the on-ledger contracts that depend on it are undermined.

## The Concept

Decentralized attestation replaces the single oracle with a protocol where multiple independent parties attest to a fact. The Canton ledger accepts the fact only when a sufficient number of attestors agree, similar to how Canton's confirmation protocol requires stakeholder consensus for transactions.

The core idea: an attestation is a signed claim by a party that some off-chain condition holds. When enough independent attestors produce consistent claims, a Daml workflow can treat the attested fact as reliable input.

## Vaults

Vaults are the planned execution environment for attestation logic. A vault is a secure enclave (or trusted execution environment) that runs attestation code in an isolated, verifiable context. The vault fetches external data, evaluates it against defined criteria, and produces signed attestations that Canton contracts can consume.

Vaults are designed to provide:

- **Isolation** -- Attestation logic runs in a protected environment, reducing the risk of tampering
- **Verifiability** -- Other parties can verify that the vault ran the expected code on the expected inputs
- **Decentralization** -- Multiple independent vaults can attest to the same fact, removing single points of trust

## Current Status

Decentralized attestation and vaults are under active development and not yet available for application developers. The concepts described here reflect the architectural direction, but APIs, SDKs, and deployment tooling are not finalized.

If your application needs off-chain data today, the standard approach is to model an attestor party in Daml that signs contracts representing external facts. You can design your contracts so that replacing a centralized attestor with decentralized attestation later requires minimal changes to your data model.
