---
title: "CanonicalCommitSet"
description: "Leaders include, with their proposal of a new block, a strong quorum of signed commit messages that completed the previous block. This is what makes this set canonical and ensures every validator agrees on it, even if it is different from the commits a validator has seen, which caused the validator to commit the respective block. Check the CometBFT documentation for the original concept: https://docs.cometbft.com/v0.37/spec/consensus/consensus#canonical-vs-subjective-commit."
---
