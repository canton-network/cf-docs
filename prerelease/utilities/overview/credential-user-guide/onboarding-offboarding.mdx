# Onboarding/Offboarding

## Request a User Service contract

After uploading the DAR file to your validator node and creating a user party, the onboarding
requires the creation of a user service contract. The creation of the service contract follows a
request and accept flow, where any party can request the establishment of a user service contract
with the operator. The acceptance of the request by the operator will trigger the contract’s
creation. Both the operator and the user are signatories on that contract.

![image](overview/credential-user-guide/pictures/request-user-service-contract.png)

## Cancel/Reject a User Service Request

The Service request can be canceled by the user (before it is accepted) or it can be rejected by the
operator, in which case no service contract will be created.

![image](overview/credential-user-guide/pictures/cancel-reject-service-contract.png)

## Termination of the User Service contract

The Service Contract can be terminated unilaterally by the operator (or by the user) in which case
the contract will be archived and the user loses access to the credential module. The credentials
owned or issued by the user stay active.

![image](overview/credential-user-guide/pictures/termination-user-service-contract.png)
