# Getting Started

## Request a Credential Service

<!-- TODO: improve this title ^^ -->

### via the UI

1. Log in to the Utility UI `https://<ingressHost.internal>` with your user set up in [Prerequisites](prerequisites.md#utility-prerequisites)
2. Select the “Credential” module
3. Press “REQUEST CREDENTIAL USER SERVICE”

Once these steps are complete, a request will appear in the corresponding DA Utility UI. To complete the onboarding, DA
will have to approve the request.

### via the API

<!-- TODO: link to the API below -->

If you did not deploy the UI, you can create a [UserServiceRequest](../daml-api-reference/credential-app-model/Utility-Credential-App-V0-Service-User.md#type-utility-credential-app-v0-service-user-userservicerequest-47604) contract directly via the API, which is described
in the [Daml API Reference](../reference/daml-api-reference/api-reference.md#daml-api-reference). It is important you set the Utility operator party correctly when creating the contract.

## Guides and Examples

* Utility [Daml API Reference](../reference/daml-api-reference/api-reference.md#daml-api-reference) contains the API documentation for the credential models, credential app, registry models and the registry app, respectively.
* Code examples using [curl](https://github.com/DACH-NY/canton-utility-examples/blob/main/doc/credential/user/manual.md) and [TypeScript](https://github.com/DACH-NY/canton-utility-examples/blob/main/app/cli/manual.md) describe how to use the API in practice, containing examples both how to write (including explicit disclosure) and how to read.

* Additionally, read the user guides for the [Credential Utility](../overview/credential-user-guide/index.md) and [Registry Utility](../overview/registry-user-guide/index.md) to gain deeper insights into specific workflows and functionalities.
